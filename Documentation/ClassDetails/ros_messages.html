ROS Message Interface:

Custom Simulation Message Pacakge

  • Package contains message definitions (for messages that do not use standard ROS messages)
  • In order to use the messages with custom types, you must include this package in your works space.
  • Repo link: https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws/tree/master/src/sim_ros_msgs

Input

  1. FollowJointTrajectory action, as used by MoveIt:
    • Type: (Action) control_msgs/FollowJointTrajectory
    • Topic: User Defined (in model)
      • "/left_arm/joint_trajectory_action" - handle left arm joint trajectory action (for yumi)
      • "/right_arm/joint_trajectory_action" - handle right arm joint trajectory action (for yumi)
    • Sim Protocol: SimMsgs/FollowJointTrajectoryService
  2. GripperCommand action, as used by MoveIt
    • Type: (Action) control_msgs/GripperCommand
    • Topic: User Defined (in model)
      • "/left_gripper/gripper_action" - handle the left hand's gripper (for yumi)
      • "/right_gripper/gripper_action" - handle the right hand's gripper (for yumi)
    • Sim Protocol: SimMsgs/GripperCommand
  3. Twist command, currently for setting the velocity target of an object
    • Type: (Action) sim_ros_msgs/TwistCommand
    • Topic: /{model_instance_name}/set_velocity_goal
    • Sim Protocol: SimMsgs/set_velocity_goal
  4. Reset command
    • Type: (Action) sim_ros_msgs/EnvReset
    • Topic: /environment/reset
    • Sim Protocol: SimMsgs/EnvironmentResetService
  5. Joint command
    • Type: (Action) sim_ros_msgs/JointCommand
    • Topic:/{model_instance_name}/joint_command
    • Sim Protocol: SimMsgs/JointCommandService

Output

  1. sensor_msgs/Image: RGB
    • Type: (Subscribe) sensor_msgs/Image
    • Topic: "/sim_cam/rgb" – capture groundtruth image in RGBA8 format
    • Topic: User Defined (in model)
    • Sim Protocol: SimMsgs/ImageData
  2. sensor_msgs/Image: Depth
    • Type: (Subscribe) sensor_msgs/Image
    • Topic: "/sim_cam/depth" – (in model) capture depth image in mono8 format: [0, 1] = [0cm, 1000cm]
    • Sim Protocol: SimMsgs/ImageData
  3. sensor_msgs/CameraInfo: Camera Info of each camera sensors (RGB, depth etc.)
    • Type: (Subscribe) sensor_msgs/CameraInfo
    • Topic:
      • "/sim_cam_info/rgb" (for rgb cam)
      • "/sim_cam_info/depth" (for depth cam)
      • can also be customized per camera basis
    • Sim protocol: SimMsgs/CameraInfoData
  4. sensor_msgs/JointState: Joint state information both for robots and for articulated objects, namespaced accordingly
    • Type: (Subscribe) sensor_msgs/JointState
    • Topic:
      • "/{model_instance_name}/joint_states" - e.g. /yumi/joint_states for model named "yumi"
      • Custom topics: user can optionally specify custom message topics in each model type. (NOTE: multiple models of the same type will publish to the same custom topic)
    • Sim Protocol: SimMsgs/JointStateData
  5. tactile sensor messages
    • Type: sim_ros_msgs/ContactPoints
    • Topic: User Defined (in model -> contact sensor component)
      • EX. for allegro hand index finger: /sensors/tactile/allegro_finger_index/contact
    • Sim Protocol: SimMsgs/ContactData
  6. tf/tfMessage: World transforms of each models in the environment
    • Type: tf/tfMessage (see http://wiki.ros.org/tf)
    • Topic: /tf
    • Sim Protocol: SimMsgs/WorldTransformsData
    • tf is in right-handed coordinate
