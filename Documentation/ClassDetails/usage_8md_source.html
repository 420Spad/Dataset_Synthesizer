4 ## NV_UE_HOME:
5 In the simulator's directory root, there are scripts that help build and launch it. All of these scripts all use an environment variable name "NV_UE_HOME". This variable need to be set to the root directory of the Unreal Engine (where the setup.sh and the GenerateProjectFiles.sh are).<br>
6 There is a script name **set_as_default_ue.sh** in the root of our version of the UnrealEngine. If you run ". set_as_default_ue.sh" it will set that UE version to be the official NV_UE_HOME<br>
7 For convenience purpose, you can edit your shell's environment and set NV_UE_HOME to the UnrealEngine folder you want to use

9 ## Simulator:
10 Repo: https://gitlab-master.nvidia.com/NVStudios_DLToolkit/Simulator<br>
11 **build_editor.sh** - build the simulator's editor. You must run this script every time after you get a new code changes or new version drop <br>
12 **launch_editor.sh** - launch the simulator's editor. With the editor, you can create and modify the maps/environments and can also launch the simulator using the Play button.<br>
13 **launch_sim.sh** [MapName] - launch the simulator. The optional parameter "MapName" is the map you want the simulator to run. If not specified, the simulator will run on map `EnvDefault`<br>

15 ## DLBroker Master:
16 The DLBroker master allow the simulator to communicate (e.g: publish messages, receive command â€¦) with external application.
17 To start the DLBroker master, run: **launch_broker.sh** in the simulator directory.<br>
18 <br>
19 NOTE:
20  * The DLBroker is built as a server so it can run for a long time while doesn't take much memory if there are no simulator or bridge connected to it.
21  * The DLBroker use port 9505 by default. You need to make sure the port is opened. More controls will be added later
22  * Right now there can be only 1 simulator connect to the DLBroker. If there are more than 1 simulators running at the same time, the message may get tangled up. More controls for multiple simulators will be added later
23  * The DLBroker must run on the same machine as the simulator

25 ## SimROSBridge:
26 Repo: https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws<br>
27 The SimROSBridge is a ROS node which communicate with the Simulator and publish, receive messages to and from other ROS nodes.<br>
28 The repo is a ROS workspace, so before you can build and run the bridge, you need to start your ROS environment (e.g: source /opt/ros/lunar/setup.bash).<br>
29 <br>
30 In the root directory, there are 2 scripts:<br>
31 **build.sh** - build the ROS node. Need to run this script after getting new code update<br>
32 **start_bridge.sh** - start the simulator-ROS bridge.<br>
33 <br>
34 NOTE:
35  * In order for the bridge to connect with the simulator, you MUST launch the DLBroker master before the ROS bridge
36  * If the simulator already running when the bridge launched, you need to restart the simulator in order for the ROS bridge to register with all the simulator's topics
37  * If the bridge is running in a different machine to the DLBroker, you need to specify the host address and port:
38  * rosrun sim_ros_bridge sim_ros_bridge.py _host_addr:=10.20.13.15 _host_port:=9505
39  * BUG: There is a bug in the ROS bridge which prevent it from shutting down by using Ctrl+C, you will need to kill its process.
