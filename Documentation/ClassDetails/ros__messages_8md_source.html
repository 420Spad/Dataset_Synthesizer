6 ## Custom Simulation Message Pacakge
7 - Package contains message definitions (for messages that do not use standard ROS messages)
8 - In order to use the messages with custom types, you must include this package in your works space.
9 - Repo link: https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws/tree/master/src/sim_ros_msgs

11 ## Input
12 1. FollowJointTrajectory action, as used by MoveIt:
13  - Type: (Action) [control_msgs/FollowJointTrajectory](http://docs.ros.org/api/control_msgs/html/action/FollowJointTrajectory.html)
14  - Topic: User Defined (in model)
15  - "/left_arm/joint_trajectory_action" - handle left arm joint trajectory action (for yumi)
16  - "/right_arm/joint_trajectory_action" - handle right arm joint trajectory action (for yumi)
17  - Sim Protocol: SimMsgs/FollowJointTrajectoryService
18 
19 2. GripperCommand action, as used by MoveIt
20  - Type: (Action) [control_msgs/GripperCommand](http://docs.ros.org/api/control_msgs/html/action/GripperCommand.html)
21  - Topic: User Defined (in model)
22  - "/left_gripper/gripper_action" - handle the left hand's gripper (for yumi)
23  - "/right_gripper/gripper_action" - handle the right hand's gripper (for yumi)
24  - Sim Protocol: SimMsgs/GripperCommand
25 
26 3. Twist command, currently for setting the velocity target of an object
27  - Type: (Action) [sim_ros_msgs/TwistCommand](https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws/blob/master/src/sim_ros_msgs/action/TwistCommand.action)
28  - Topic: /{model_instance_name}/set_velocity_goal
29  - Sim Protocol: SimMsgs/set_velocity_goal
30 
31 4. Reset command
32  - Type: (Action) [sim_ros_msgs/EnvReset](https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws/blob/master/src/sim_ros_msgs/action/EnvReset.action)
33  - Topic: /environment/reset
34  - Sim Protocol: SimMsgs/EnvironmentResetService
35 
36 5. Joint command
37  - Type: (Action) [sim_ros_msgs/JointCommand](https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws/blob/master/src/sim_ros_msgs/action/JointCommand.action)
38  - Topic:/{model_instance_name}/joint_command
39  - Sim Protocol: SimMsgs/JointCommandService

42 ## Output
43 1. sensor_msgs/Image: **RGB**
44  - Type: (Subscribe) [sensor_msgs/Image](http://docs.ros.org/api/sensor_msgs/html/msg/Image.html)
45  - Topic: "/sim_cam/rgb" -- capture groundtruth image in RGBA8 format
46  - Topic: User Defined (in model)
47  - Sim Protocol: SimMsgs/ImageData
48 
49 2. sensor_msgs/Image: **Depth**
50  - Type: (Subscribe) [sensor_msgs/Image](http://docs.ros.org/api/sensor_msgs/html/msg/Image.html)
51  - Topic: "/sim_cam/depth" -- (in model) capture depth image in mono8 format: [0, 1] = [0cm, 1000cm]
52  - Sim Protocol: SimMsgs/ImageData
53 
54 3. sensor_msgs/CameraInfo: **Camera Info** of each camera sensors (RGB, depth etc.)
55  - Type: (Subscribe) [sensor_msgs/CameraInfo](http://docs.ros.org/kinetic/api/sensor_msgs/html/msg/CameraInfo.html)
56  - Topic:
57  - "/sim_cam_info/rgb" (for rgb cam)
58  - "/sim_cam_info/depth" (for depth cam)
59  - can also be customized per camera basis
60  - Sim protocol: SimMsgs/CameraInfoData
61 
62 4. sensor_msgs/JointState: **Joint** state information both for robots and for articulated objects, namespaced accordingly
63  - Type: (Subscribe) [sensor_msgs/JointState](http://docs.ros.org/api/sensor_msgs/html/msg/JointState.html)
64  - Topic:
65  - "/{model_instance_name}/joint_states" - e.g. /yumi/joint_states for model named "yumi"
66  - Custom topics: user can optionally specify custom message topics in each model type. (NOTE: multiple models of the same type will publish to the same custom topic)
67  - Sim Protocol: SimMsgs/JointStateData
68 
69 5. tactile sensor messages
70  - Type: [sim_ros_msgs/ContactPoints](https://gitlab-master.nvidia.com/NVStudios_DLToolkit/sim_ros_bridge_ws/blob/master/src/sim_ros_msgs/msg/ContactPoints.msg)
71  - Topic: User Defined (in model -> contact sensor component)
72  - EX. for allegro hand index finger: /sensors/tactile/allegro_finger_index/contact
73  - Sim Protocol: SimMsgs/ContactData
74 
75 6. tf/tfMessage: **World transforms** of each models in the environment
76  - Type: [tf/tfMessage](http://docs.ros.org/api/tf/html/msg/tfMessage.html) (see http://wiki.ros.org/tf)
77  - Topic: /tf
78  - Sim Protocol: SimMsgs/WorldTransformsData
79  - tf is in right-handed coordinate
